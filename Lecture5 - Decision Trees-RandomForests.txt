Decision Trees: Core Idea

Start with all training data.

Choose a feature and a threshold.

Split the data into two groups.

Repeat recursively on each branch.

Stop when further splitting is unnecessary.

The result is a tree of decisions.

Decision trees are a type of classifier network.

They are very easy to interpret which means that if causal reasoning is important
then they can be one of the better ways to go.

This works because it is very easy to trace our way back up the tree to determine
why each decision was made.

In general, trees are upside down with the root at the top and the leaves at the
bottom.

Nodes that are neither the root or a leaf are called internal nodes. If we get to a
leaf node, a classification has been made.

How Does a Tree Decide Where to Split?

At each node, the algorithm chooses the split that produces the purest child nodes.
Purity measures how mixed the class labels are after the split.
Common impurity measures:

Gini impurity (default in scikit-learn)

Entropy

Both are a measure of how ”pure” a branch is. If we can split so that 100% of
values on each side are in one category, then that’s good. If we split so that every
outcome is equally likely on each side, that’s bad. Create the decision that
maximizes ”goodness”.

Either method seems to produce similar trees. In either case, we create a decision,
and then recurse until we have to stop.

Ensembles

It can sometimes be hard to balance the complexity of the classifier with
over-fitting. Sometimes, we’ll find that no single classifier will do a great job on
the problem.

It can be easy to produce a classifier that makes predictions better than chance,
but still not great.

Often useful to have many simple classifiers, with some differences in their
construction so they produce different results and let them vote: an ensemble

An ensemble can often produce good results, even though the individual classifiers
are quite bad.

Combining many weak models can give a much stronger and robust result that
generalises better.

The wisdom of the crowd

Random Forests

A random forest is an ensemble of decision trees where:

Each tree is trained on a bootstrap sample

Each split considers only a random subset of features

Predictions are aggregated by voting

. . . and repeat many times.

Random Forest Algorithm

1 Draw a random bootstrap sample of size n (randomly choose n samples from the
training set with replacement i.e. there can be duplicates).

2 Grow a decision tree from the bootstrap sample. At each node:
Randomly select d features without replacement i.e. no duplicates.
Split the node using the feature that provides the best split according to the
objective function, for instance, by maximising the information gain.

3 Repeat the steps 1 and 2 k times.
4 Aggregate the prediction by each tree to assign the class label by majority vote.

Reproducibility with Random Forests and Cross Validation
Random forests contain randomness:

Bootstrap sampling of training data
Random subset of features at each split

To ensure consistent cross-validation results, fix the random seed:

from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score

model = RandomForestClassifier(
n_estimators =100,
random_state =42

)

%scores = cross_val_score(model , X_train , y_train)

Setting random state ensures reproducible results across runs.

Multi-class Classification

The last two model types we studied were kNN and RandomForest. These both
naturally handle multiple classes.

Logistic Regression and Support Vector Machines are binary classifiers at their core.

They are designed to separate two classes.
But in the lab we used LogisticRegression to classify multiple things.
How is that possible if it is a binary model?
We need strategies to extend binary classifiers to multi-class problems.

There are three main strategies:

One-vs-Rest (OvR)

One-vs-One (OvO)

Direct Multinomial Model (Softmax)

OvR and OvO are reduction strategies:

Reduce a multi-class problem to many binary problems

Softmax is a direct multi-class model.

Turn the k-class problem into k
binary problems.

Each classifier learns:

Class i vs All Other Classes

This produces k models.

Three classes: Road, Road Sign, Other.

Using One-vs-Rest

To classify a new sample:

Run it through all k classifiers.

Each classifier produces a score or probability.

Choose the class with the highest score.

OvR is now the default strategy for many SVM implementations.

Donny Hurley Decision Trees 48 / 55



Properties of One-vs-Rest

Requires k classifiers.

Simple and widely used.
Can suffer from class imbalance:

One positive class
Many negatives

Works well in practice.

Modern SVM implementations typically use OvR by default.
