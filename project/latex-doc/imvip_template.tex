\begin{abstract}
This study explores the development of a deep learning system to classify 19 different types of food. Recognising food from images is a difficult task because many dishes look very similar and the way a meal is plated can vary significantly. This research compares two main methods: a custom-built Convolutional Neural Network (CNN) and a Transfer Learning approach that uses a pre-trained model as its foundation.
\end{abstract}
\textbf{Keywords:} Imaging, Image Processing, Machine Vision, etc. (Maximum five)

\section{Introduction}

Initial attempts with a custom model resulted in clear underfitting, where the accuracy was no better than random guessing at 5.2\%. By improving the model structure, specifically by using double convolutional blocks, increasing the number of filters to 128, adding Dropout Layers to prevent over-reliance on specific features, the custom model eventually achieved a stable validation accuracy of 65\%. To try and improve further, a two-phase transfer learning strategy was used. This involved an initial phase where the base model was frozen, followed by a fine-tuning phase to adapt the weights to this specific food dataset.

\section{Data and Pre-Processing}
The first thing I did was to initialise any of the variables I will be using that will detail how I will be processing my data. The seed that I set is \texttt{421944} for my student number, this will allow for reproducability of my results. I then have the batch sizes for my data in batches of \texttt{32}, dealing with image sizes of \texttt{128x128} and \texttt{64x64}. Computational tasks were performed locally using an NVIDIA GeForce RTX 4060 GPU, with a verification step included in the script to confirm CUDA acceleration was active.

\subsection{Dataset Description}
The image dataset being used is a subset of the Food 101 dataset containing 19 folders named after the type of food contained in its images. The images in use are \texttt{512x512}, but will be rescaled across \texttt{128x128} and \texttt{64x64}, grayscaled and/or augmented for the purpose of exploring different hyperparameters and comparing models.

\subsection{Preprocessing and Splits}

The dataset contained images with 3 colour channels (RGB), since that would involve numbers ranging from 0.0 to 255, it was important the training data was normalised. Normalisation will help with data moving between layers, and help prevent vanishing/exploding gradients. The training split that was chosen was to split the Training Set at 30\% (13,312 images), Validation 10\% (1,888 images), for tuning the hyperparameters and the final Test Set at 20\% (3,840 images). The final Test Set is reserved to the final evaluation of the model.

\begin{itemize}
  \item Training Set at 30\% (13,312 images) - Used for model weight updates
  \item Validation 10\% (1,888 images) - Hyperparameter Tuning
  \item Test Set at 20\% (3,840 images) - Reserved to the final evaluation of the model.
\end{itemize}

\subsection{Exploratory Analysis}

An initial audit of the dataset confirmed it is perfectly balanced. Each of the 19 classes contains exactly $1,000$ images, totaling $19,000$ samples with 3 channels and 128x128 in size. This eliminates the risk of class imbalance, where a model might become biased toward a majority class to artificially inflate accuracy. In such a case where there was an imbalance, class weights can be introduced to give equal importance to all classes.

\section{Experiments}

\subsection{CNN From Scratch: Iterative Development}

The development of the custom Convolutional Neural Network (CNN) followed an iterative progression, transitioning from a high-parameter baseline to a refined, efficient architecture optimised for generalisation on my dataset.

\subsubsection{Initial Models: From Overfitting to Underfitting}
The development began with \textbf{Model V1 (Baseline)}, which utilised two convolutional blocks and a single Max Pooling layer for downsampling. This model exhibited extreme overfitting, achieving a training accuracy of \textbf{98.32\%} while validation accuracy stagnated at \textbf{16.84\%}. This massive disparity indicated that the network was memorising noise within the training set rather than learning generalisable features.

To mitigate this, \textbf{Model V2} introduced three convolutional blocks with doubled layers per block to extract more granular features. A dense layer of 256 units and a Dropout rate of 0.5 were added to reduce feature dependency. However, this resulted in an over-correction into underfitting, with both training and validation accuracies dropping below \textbf{6\%}.

\subsubsection{Structural Refinement and Regularisation}
\textbf{Model V3} aimed to find a middle ground by reverting to two convolutional blocks while introducing \textbf{Batch Normalization} to stabilize internal covariate shift. A significant architectural change was the replacement of the flattening layer with \textbf{GlobalAveragePooling2D}, which reduced the parameter count from approximately \textit{8.6 million} to just \textit{41,299}. This led to smoother convergence and a more aligned relationship between training and validation metrics.

In \textbf{Model V4}, \textbf{Image Augmentation} \textit{(horizontal flips, 10\% rotations, and 10\% zooms)} was integrated into the pipeline to artificially expand the dataset diversity. While this stabilized the training loss, the model remained sensitive to extreme transformations, resulting in a validation accuracy of approximately \textbf{22.99\%}. However, it was at this point that I noticed the image size \textbf{64x64} had outperformed \textbf{128x128} having reached \textbf{34.80\%} validation accuracy. This led me to investigate that having only two convolutional layers for a \textbf{128x128} image wasn't enough for the model to correctly learn. The rescaling of the image to \textbf{64x64} actually helped remove some of that extra noise, paired with the augementation, actually performed better.

\subsubsection{Scaling and Final Optimisation}
\textbf{Model V5} followed with increase in validation reaching an accuracy of \textbf{62.92\%} with the addition of extra two convolutional layers to learn more complex features. The usage of \textbf{Batch Normalisation} on each of those layers stablised the learning progress even further, allowing for me to increase my learning rate back up to $1 \times 10^{-2}$.

Over-engineering became the downfall in validation, to \textbf{44.81\%} and training accuracy, to \textbf{47.58\%} with \textbf{Model V6} due to drastically increasing the width and depth (adding 512 and 1024-filter layers) and an enormous 2048-unit Dense layer. A layer with 1024 filters and a dense layer of 2048 showcased that the model was memorising specific noise rather than learning general features. By the time the data reaches the end of V6, the spatial resolution is tiny, but the number of channels is massive. The GlobalAveragePooling2D had to compress 1024 channels, which can lead to a loss of important spatial relationship info.

Finally, \textbf{Model V7} was developed to prioritise efficiency over raw size. By reducing the dense layer from 2048 to 128 units and implementing a \texttt{ReduceLROnPlateau} scheduler, the model achieved a \textbf{32\% reduction in validation loss} compared to V6. V7 demonstrated superior generalisation by maintaining a validation accuracy of \textbf{63.77\%}.

\subsection{Transfer Learning Analysis}

Transfer learning was implemented to evaluate if a model pre-trained on a large-scale dataset (ImageNet) could outperform the custom CNN architectures previously developed. This approach allows the model to leverage complex feature extractors that have already learned generic visual patterns like edges, textures, and shapes.

\subsubsection{Model Selection: MobileNetV2}
MobileNetV2 \cite{tf_mobilenetv2} was chosen as the base architecture over larger models like VGG16 or ResNet50 for several key reasons:
\begin{itemize}
    \item \textbf{Efficiency:} MobileNetV2 is designed for mobile and resource-constrained environments, making it highly efficient for the 128x128 input size used in this project.
    \item \textbf{Parameter Economy:} The base model contains \textbf{2,257,984 parameters}, yet it allows for a "frozen" state where \textbf{0 parameters} are trainable during the initial phase. This prevents the destruction of pre-trained weights while training the new classification head.
    \item \textbf{Performance vs. Size:} It provides a high accuracy-to-parameter ratio, which is critical for maintaining fast training times without sacrificing depth.
\end{itemize}

\subsubsection{Phase 1: Feature Extraction}
In the first phase, the MobileNetV2 base is used strictly as a feature extractor. The following architecture was implemented:

\begin{itemize}
    \item \textbf{Image Size:} The results shown in Table~\ref{tab:model_performance_grayscaled} showcase that in our chosen CNN Model that the image size \textbf{128x128} outperformed \textbf{64x64} in both training and accuracy results.
    \item \textbf{Preprocessing:} The \texttt{preprocess\_input} function is critical as it scales input pixels to the range $[-1, 1]$, which is the specific requirement for MobileNetV2 to function correctly.
    \item \textbf{Frozen Base:} The base model is called with \texttt{training=False}, ensuring that Batch Normalization layers continue to use the statistics learned from ImageNet rather than the small batch statistics of the current dataset.
    \item \textbf{Global Average Pooling (GAP):} Instead of a Flatten layer, GAP was used to reduce the spatial dimensions of the feature maps, significantly reducing the parameter count in the dense layers and helping to prevent overfitting.
    \item \textbf{Custom Head:} A dense layer of 128 units with ReLU activation was added, flanked by two \textbf{Dropout layers (0.3)} to ensure robust regularization.
\end{itemize}

Using a standard learning rate of $1 \times 10^{-3}$, this phase achieved a \textbf{best validation accuracy of 71.61\%}.

\subsubsection{Phase 2: Fine-Tuning}
To further optimise performance, a fine-tuning phase was initiated. The top layers of the base model were unfrozen to allow them to adapt to the specific nuances of the food dataset.

\begin{itemize}
    \item \textbf{Selective Unfreezing:} Out of the \textbf{154 total layers} in MobileNetV2, only the \textbf{top 20 layers} were made trainable. This keeps the early "general" feature detectors intact while specializing the deeper "complex" detectors.
    \item \textbf{Learning Rate Strategy:} The learning rate was reduced to \textbf{$1 \times 10^{-5}$}. This is essential to avoid large weight updates that could degrade the pre-trained feature representations.
\end{itemize}

During this phase, the validation accuracy reached \textbf{72.72\%}. While slightly higher than Phase 1, the training curves in [CITE HERE] show that the validation loss remained stable around \textbf{0.88\%}, indicating the model was well-regularised.

\section{Real-World Evaluation}

To assess the \textbf{practical utility} of the final Transfer Learning model (MobileNetV2 Phase 2), the system was tested on novel images captured in a non-laboratory environment. Unlike the standardized Food-101 dataset, these images contain varied lighting, complex backgrounds (e.g., handwritten notes), and non-standard plating.

\subsection{Qualitative Analysis of Predictions}

The model demonstrated a robust ability to generalize, successfully identifying the primary food items despite the presence of "visual noise" in the background.

\subsection{Confidence and Uncertainty}

Analysis of the prediction probabilities reveals key insights into the model's decision-making process:

\begin{itemize}
    \item \textbf{High Confidence Scenarios:} For the "Pizza" sample, the model achieved a high confidence score of \textbf{89.0\%}. The distinct geometric features of the pepperoni and the triangular crust allowed for clear separation from secondary classes like \textit{spaghetti\_bolognese} (4.4\%).
    \item \textbf{Ambiguity and Feature Overlap:} The "Garlic Bread" sample yielded a lower confidence score of \textbf{41.9\%}. Interestingly, the top alternative was \textit{cheesecake} (16.3\%). This confusion likely stems from the shared chromatic profile (pale yellows and whites) and the presence of the white plate, which the model may have associated with dessert presentation patterns found in the training set.
\end{itemize}

Despite these ambiguities, the correct class remained the top prediction in both instances, confirming that the \textbf{fine-tuning phase} successfully adapted the ImageNet weights to capture the specific nuances of the 19 food categories.

\section{Conclusion}

This research successfully demonstrated the iterative development of a deep learning system for 19-class food classification, navigating the complexities of high intra-class variance and inter-class similarity. The experimental results highlight a clear progression: initial custom architectures (Models V1--V4) struggled with the balance between capacity and regularisation, while structural refinements in Model V7 established a robust custom benchmark with a validation accuracy of 63.77\% and a significant 32\% reduction in loss compared to over-engineered predecessors.

The comparative analysis revealed that while custom CNNs can achieve respectable performance through careful tuning of dropout, batch normalisation, and learning rate scheduling, they are ultimately surpassed by transfer learning techniques. By leveraging MobileNetV2, the system achieved an overall best validation accuracy of 72.72\%. This success underscores the value of pre-trained feature extractors in domains where datasets are relatively small, as the model can utilise sophisticated geometric and textural filters learned from the broader ImageNet corpus.

Real-world testing further validated the practical utility of the final model, which maintained correct classifications despite the presence of non-standard backgrounds and varied lighting. While some ambiguity remains between visually similar categories—such as dough-based textures in pizza and garlic bread—the implementation of a top-3 probability output provides a reliable framework for handling predictive uncertainty. Future work could explore higher-resolution inputs or the integration of attention mechanisms to further resolve the fine-grained details necessary for professional culinary recognition.
